# Environment Variables

Required for production:

- DATABASE_URL
- NEXTAUTH_URL
- NEXTAUTH_SECRET
- AWS_REGION
- AWS_ACCESS_KEY_ID
- AWS_SECRET_ACCESS_KEY
- AWS_S3_BUCKET
- AWS_CLOUDFRONT_URL
- SEGPAY_WEBHOOK_SECRET or processor equivalent

Recommended:

- APP_ENV
- ADMIN_EMAILS
- SUPPORT_EMAIL
- RATE_LIMIT_WINDOW_SECONDS
- RATE_LIMIT_MAX_REQUESTS
 - LOG_LEVEL

## Optional authentication variables

These variables enable third‑party sign‑in providers powered by [NextAuth.js](https://next-auth.js.org/). They are not required for a basic deployment, but if you wish to allow fans and creators to log in with Google or Apple accounts you must supply the appropriate credentials.

- `GOOGLE_CLIENT_ID` – OAuth client ID obtained from the Google Cloud Console. This must correspond to a Web application with an authorised redirect URI pointing at your site (e.g. `https://yourdomain.com/api/auth/callback/google`).
- `GOOGLE_CLIENT_SECRET` – OAuth client secret matching the above client ID.
- `NEXT_PUBLIC_GOOGLE_AUTH_ENABLED` – set to `"true"` to display the “Continue with Google” button on the login and signup pages. When this flag is `"true"` and the client ID/secret are defined, the Google provider is automatically enabled by the authentication handler.
- `APPLE_CLIENT_ID` – your Apple Services ID or App ID registered for Sign in with Apple. Apple mandates a production domain – you cannot use `localhost` for Apple sign‑in【736170990043017†L166-L167】.
- `APPLE_CLIENT_SECRET` – a JSON Web Token (JWT) generated by your Apple developer credentials. NextAuth expects this secret to be pre‑signed and valid for at least a day. Consult Apple’s documentation or use tools like `next-auth/apple-gen-secret` to generate this value【736170990043017†L131-L147】.
- `NEXT_PUBLIC_APPLE_AUTH_ENABLED` – set to `"true"` to show the “Continue with Apple” button. The sign‑in flow will only succeed when `APPLE_CLIENT_ID` and `APPLE_CLIENT_SECRET` are also configured.
