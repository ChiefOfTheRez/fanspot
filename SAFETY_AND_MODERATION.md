# Safety and Moderation Plan

## Moderation goals

FanSpot should not rely on manual judgment after problems happen. The platform should have clear review queues, reporting tools, audit logs, and safe defaults from the beginning.

## MVP safety controls

- Reports table
- Moderation status on posts
- Admin report queue
- Content review shell
- Creator application review
- Audit logs
- Uploads disabled by default until S3 and moderation are ready

## Suggested content statuses

- DRAFT
- PENDING_REVIEW
- APPROVED
- REJECTED
- FLAGGED
- REMOVED

## Suggested report reasons

- Spam
- Harassment
- Impersonation
- Unsafe content
- Payment issue
- Copyright/trademark issue
- Other

## Admin actions

- Approve
- Reject
- Hide
- Remove
- Request more information
- Suspend account
- Restore account
- Escalate to legal/compliance

## Before launch

Add:

- Real report submission form
- Admin action logging
- Appeal workflow
- Email notifications
- Creator policy acknowledgement
- Terms and Privacy reviewed by a lawyer
